## 📂 文件权限的结构

在终端中用 `ls -l` 查看文件，会看到类似这样的输出：

```
-rwxr-xr--
```

它表示一个文件的权限，分成 10 个字符，我们逐段解释：

```
[1]   [2][3][4]
-     rwx r-x r--
```

### 解释：

| 字符位置   | 含义              |
| ------ | --------------- |
| 第 1 位  | 文件类型（- 文件，d 目录） |
| 2-4 位  | 所有者（user）的权限    |
| 5-7 位  | 所在组用户（group）的权限 |
| 8-10 位 | 其他人（others）的权限  |

---

## 🔐 权限类型

| 字母 | 含义            | 针对文件 | 针对目录     |
| -- | ------------- | ---- | -------- |
| r  | 读权限（read）     | 查看内容 | 列出目录文件名  |
| w  | 写权限（write）    | 修改内容 | 创建/删除文件  |
| x  | 执行权限（execute） | 执行文件 | 进入目录（cd） |
| -  | 没有该权限         | 无    | 无        |

---

## 🧑 三种用户身份

| 身份类型        | 说明         |
| ----------- | ---------- |
| 所有者（user）   | 文件的拥有者     |
| 所属组（group）  | 文件归属的用户组   |
| 其他人（others） | 系统中的所有其他用户 |

---

## ✅ 举例说明

```
-rwxr-xr--
```

表示：

* `-`：是一个普通文件（不是目录）
* `rwx`：所有者有**读、写、执行**权限
* `r-x`：所属组有**读、执行**权限
* `r--`：其他用户只有**读**权限

---

## 🎯 权限的数字表示法（八进制）

| 权限    | 二进制 | 八进制 |
| ----- | --- | --- |
| `---` | 000 | 0   |
| `--x` | 001 | 1   |
| `-w-` | 010 | 2   |
| `-wx` | 011 | 3   |
| `r--` | 100 | 4   |
| `r-x` | 101 | 5   |
| `rw-` | 110 | 6   |
| `rwx` | 111 | 7   |

### 示例：

* `chmod 755 filename` 相当于：

  * 所有者：`7 = rwx`
  * 所属组：`5 = r-x`
  * 其他人：`5 = r-x`

---

## 🛠 常用命令

| 命令                 | 功能              |
| ------------------ | --------------- |
| `ls -l`            | 查看文件权限          |
| `chmod +x file`    | 给文件添加执行权限       |
| `chmod 644 file`   | 设置权限（rw-r--r--） |
| `chown user file`  | 更改文件所有者         |
| `chgrp group file` | 更改文件所属组         |

---

## 🔒 示例理解

* `rw-r--r--` → 拥有者可以读写，组和其他人只能读
* `rwx------` → 只有拥有者可以读写执行，其他人完全禁止
* `rwxrwx---` → 拥有者和组都可以完全操作，其他人禁止访问
