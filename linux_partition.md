# 🧭 Linux 分区与挂载整理笔记

---

## 🧱 一、Linux 中的常见分区类型（按用途）

| 分区名              | 挂载点    | 作用                                               | 是否必须       |
| ---------------- | ------ | ------------------------------------------------ | ---------- |
| `/`（根分区）         | 根目录    | Linux 系统的核心目录，包含所有系统目录（如 `/bin`, `/etc`, `/usr`） | ✅ 必须       |
| `/boot`（引导分区）    | 启动引导目录 | 存放引导加载器（如 GRUB）、内核文件                             | 推荐单独分区     |
| `/home`          | 用户主目录  | 存储用户数据和配置文件                                      | 可选（建议单独分区） |
| `/var`           | 可变数据目录 | 存储日志、缓存、邮件等                                      | 可选         |
| `/usr`           | 用户应用程序 | 包含大多数用户工具和库文件                                    | 可选         |
| `/tmp`           | 临时文件   | 系统或程序运行时生成的临时文件                                  | 可选         |
| `swap`（交换分区）     | 无挂载点   | 作为虚拟内存使用                                         | 推荐存在       |
| `/mnt`, `/media` | 挂载目录   | 临时挂载其他设备（如U盘、光盘）                                 | 可选         |

---

## 🔍 二、分区类型（按结构）

| 类型名称                         | 说明                 |
| ---------------------------- | ------------------ |
| **基本分区（Primary Partition）**  | 一个硬盘最多只能有 4 个主分区   |
| **扩展分区（Extended Partition）** | 特殊的主分区，用于容纳多个逻辑分区  |
| **逻辑分区（Logical Partition）**  | 创建在扩展分区中，突破主分区数量限制 |

---

## ⚙️ 三、各分区作用简述

| 分区名              | 描述                 |
| ---------------- | ------------------ |
| A. 基本分区          | 最基本的磁盘分区类型，最多有4个   |
| B. 交换分区（Swap）    | 提供虚拟内存支持，不挂载到文件系统  |
| C. 根分区（`/`）      | 系统的核心分区，必须存在       |
| D. 引导分区（`/boot`） | 存放启动引导程序和内核，便于系统启动 |

---

## 🗃️ 四、挂载（Mount）基础

* 挂载是将分区（设备）绑定到文件系统目录树上的过程。
* 挂载点是一个目录，例如 `/home`, `/mnt/usb`。
* 每个分区都必须挂载到某个目录上，才能被访问。

### 常用命令：

```bash
lsblk           # 查看分区和挂载点
df -h           # 显示磁盘使用情况
mount           # 查看当前挂载状态
umount /mnt/usb # 卸载挂载点
```

---

## 🧰 五、安装时常见分区方案示例

| 分区      | 大小建议          | 挂载点     |
| ------- | ------------- | ------- |
| `/boot` | 512 MB - 1 GB | `/boot` |
| `swap`  | 与内存大小相同或略小    | 无挂载点    |
| `/`     | 20 GB 以上      | `/`     |
| `/home` | 剩余空间全部        | `/home` |

---

## ✅ 六、总结记忆要点

* **根分区 `/` 是必须的**，没有它系统无法启动。
* **交换分区不是挂载点，但非常重要**，尤其在内存不足时。
* **引导分区 `/boot` 推荐单独分**，便于引导程序的更新和管理。
* **基本分区是磁盘结构的一部分，而不是某个目录的功能。**
