进程（Process）是操作系统中正在运行的程序的实例。在操作系统中，进程通常具有以下几种**基本状态**，以及这些状态之间的**状态转换关系**：

---

## 🧩 一、进程的五种基本状态：

| 状态名称   | 英文名称              | 描述                        |
| ------ | ----------------- | ------------------------- |
| **新建** | New / Created     | 进程正在被创建（尚未投入运行）           |
| **就绪** | Ready             | 等待被调度器分配 CPU 执行           |
| **运行** | Running           | 正在使用 CPU 执行指令             |
| **阻塞** | Blocked / Waiting | 正在等待某事件（如I/O、信号）完成，无法继续执行 |
| **终止** | Terminated        | 进程已执行完毕或被强制终止，正在清理资源      |

---

## 🔄 二、状态切换图示和说明：

```text
  +------+
  | 新建 |-------------------+
  +------+                   |
     |                       v
     v                  +---------+
  创建完成              |  就绪   |
                       +---------+
                            |
                            | 调度器选择
                            v
                       +---------+
                       | 运行中  |
                       +---------+
                            |
          +----------------+------------------+
          |                                   |
          v                                   v
      时间片到/抢占                    请求I/O或等待事件
          |                                   |
          v                                   v
      +---------+                        +----------+
      |  就绪   |<----------------------+  阻塞中  |
      +---------+          I/O完成等      +----------+
                                           |
                                           v
                                       +--------+
                                       |  就绪  |
                                       +--------+

           运行中
              |
              v
         +------------+
         | 终止/结束  |
         +------------+
```

---

## 📘 三、各状态切换解释：

| 从 → 到   | 原因                          |
| ------- | --------------------------- |
| 新建 → 就绪 | 进程创建完毕，等待调度                 |
| 就绪 → 运行 | 操作系统调度进程获得 CPU              |
| 运行 → 阻塞 | 进程请求 I/O 或等待某事件（如键盘输入、网络响应） |
| 阻塞 → 就绪 | 等待的事件完成，进程准备继续运行            |
| 运行 → 就绪 | 时间片用完，被操作系统抢占               |
| 运行 → 终止 | 正常退出或被强制结束                  |
| 阻塞 → 终止 | 阻塞状态中被强制终止（少见）              |

---

## 🧠 拓展：现代系统中的更多状态（某些操作系统中）

* **挂起状态（Suspended）**：指进程被暂停执行、放到外存中（比如为了节省内存）。

  * 如：**就绪挂起、阻塞挂起**
* **僵尸态（Zombie）**：进程已经终止，但其父进程尚未调用 `wait()` 回收其信息。
